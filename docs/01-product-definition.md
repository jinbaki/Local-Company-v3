# V3 제품 정의

## 한 줄 정의

Local Company V3는 Codex가 MCP로 호출하는 로컬 AI 회사 실행 엔진이다.

## 제품 목적

대표가 Codex에게 "이 일은 PM에게 맡겨서 쫙 뿌려줘"라고 말하면, Codex는 MCP를 통해 Local Company에 일을 위임한다. Local Company의 PM은 목표를 정리하고, 필요한 작업과 산출물을 설계하고, Codex CLI worker를 실행해 산출물을 만든다. 작업이 끝나거나 대표 판단이 필요해지면 Codex가 사용자에게 알려준다.

## 핵심 사용자 경험

사용자는 Local Company 앱 안에서 PM과 채팅하지 않는다. 사용자는 Codex와 대화한다.

웹앱은 다음을 보여주는 상황판이다.

- 지금 어떤 캠페인이 실행 중인지
- PM이 어떤 계획을 세웠는지
- 어떤 worker가 어떤 일을 하고 있는지
- 산출물이 어디까지 만들어졌는지
- 대표 확인이 필요한 항목이 있는지
- Codex가 다시 사용자에게 알려야 할 새 알림이 있는지

## V3 제품 원칙

1. Codex가 조작한다.
   - 사람의 주요 입력 채널은 Codex 대화다.
   - 웹 UI는 조작보다 관찰을 우선한다.

2. PM이 설계한다.
   - 규칙 기반으로 팀과 작업을 자동 생성하지 않는다.
   - PM이 JSON 계약 안에서 목표, 작업, 산출물, worker, 결정 요청을 설계한다.

3. Local Company가 실행한다.
   - PM 계획은 검증된 뒤 작업 큐가 된다.
   - 실행 엔진이 Codex CLI worker를 순차 또는 병렬로 실행한다.

4. 산출물이 원본이다.
   - 대화가 아니라 산출물, 리뷰, 결정, 이벤트가 공식 상태다.
   - 모든 산출물은 버전으로 저장한다.

5. 필요한 순간만 사용자에게 돌아온다.
   - 완료, 실패, 대표 결정 필요, 실행 한도 도달 시 Codex가 알려준다.
   - 단순 진행 중 상태는 상황판과 알림 목록에 남긴다.

## V3에서 하지 않는 것

- 웹앱 안에서 PM 채팅을 핵심 기능으로 제공하지 않는다.
- 사람이 버튼을 눌러 작업을 하나씩 실행하는 흐름을 핵심으로 두지 않는다.
- "인계 보고서"라는 애매한 용어를 쓰지 않는다.
- 공개 버전에서 특정 사용자의 Codex 계정, 토큰, 로컬 경로를 포함하지 않는다.
- 외부 서버에 Local Company를 공개하지 않는다.

## 성공 기준

다음 문장이 실제로 가능해야 한다.

```text
Codex, 이 자료를 Local Company PM에게 맡겨서 작업을 쫙 뿌리고,
산출물이 나오면 나한테 알려줘.
```

Codex는 이 요청을 받아 MCP로 참고자료를 저장하고, 작업을 위임하고, 실행을 시작하고, 완료 알림을 보고 사용자에게 산출물을 함께 검토하자고 알려야 한다.
