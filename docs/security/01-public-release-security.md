# V3 공개 저장소 보안 기준

## 공개 저장소에 포함하지 않는 것

- `.env`
- SQLite DB
- `data/`
- 실행 로그
- Codex 인증 정보
- API key
- 사용자 로컬 경로
- 사용자 첨부 파일 원본
- IP, 계정 ID, 토큰

## 공개 저장소에 포함할 수 있는 것

- `.env.example`
- 샘플 데이터
- 문서
- 소스 코드
- 설치 스크립트
- MCP 등록 스크립트

## Codex 계정 원칙

V3는 저장소 소유자의 Codex 계정을 공유하지 않는다.

다른 사용자는 자기 PC에서 다음을 수행한다.

1. 저장소 클론
2. 의존성 설치
3. Codex CLI 설치
4. 본인의 Codex/ChatGPT 계정으로 로그인
5. MCP 등록 스크립트 실행

각 사용자의 Codex 인증은 각자의 PC에 남는다.

## Local Server 원칙

- 서버는 `127.0.0.1`에만 바인딩한다.
- 외부에서 접근 가능한 host를 기본값으로 두지 않는다.
- 공개 서버 배포는 V3 초기 범위가 아니다.

## MCP 보안 원칙

- 삭제 도구는 초기 공개 버전에 넣지 않는다.
- 파일 임의 읽기 도구를 제공하지 않는다.
- 참고자료는 Codex가 명시적으로 전달한 content만 저장한다.
- 산출물 조회는 Local Company가 만든 artifact 범위로 제한한다.
- 외부 행동, 결제, 연락, 법무 판단은 decision으로 멈춘다.

## 공개 전 검사

공개 전 자동 검사는 다음을 확인해야 한다.

- `.env`가 Git 대상이 아님
- `data/`가 Git 대상이 아님
- Windows 절대 경로가 공개 문서에 없음
- API key 패턴 없음
- Codex token 패턴 없음
- 샘플 데이터에 개인 이름, 계정, 실제 파일 경로 없음
